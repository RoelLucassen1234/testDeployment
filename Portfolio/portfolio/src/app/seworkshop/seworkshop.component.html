Inleiding

Op 7-9-2020 heb ik een workshop gekregen over het design van security. In dit document zullen we de volgende onderwerpen bespreken: secure development lifecycle, Zero trust model, devops security by design, how to establish security requirement


Secure Development Lifecycle

De secure development heeft meerdere stappen om een veilige lifecycle te maken. Ik zal in dit geval de SDL van microsoft gebruiken als voorbeeld over wat een secure development cycle kan inhouden. De Microsoft SDL heeft zeven stappen waarvan ik er vijf zal gaan bespreken.

(Stap 1) Requirements:

  • Het maken van Security Requirements. Deze worden gebruikt om te kijken wat er veilig gemaakt moet gaan worden. Dit wordt gedaan door te kijken hoe de beveiliging belangrijk is voor een systeem, wat voor data gebruikt wordt door een systeem, welke user roles er zijn en de CRUD-permissions, de CIA and privacy en Privacy impact Analysis
  • Security & Privacy Risk Assessment.

(Stap 2) Design

  • Het maken van Design Requirements
  • Threat modeling

(Stap 3) Implementation

  • Het gebruik van Static analysis om code te verbeteren en beveiligen.
  • Het gebruik van tools die veilig zijn.
  • Het verwijderen van onveilige functies.

(Stap 4) Verification

  • Dynamische analyse
  • Fuzz testing

 

(Stap 5) Release

  • Final security review om alles te testen
