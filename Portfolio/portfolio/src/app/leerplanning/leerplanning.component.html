Inleiding

In dit document zal je het leerplan van Roel Lucassen zien. Dit leerplan is wat Roel Lucassen wil gaan leren dit semester. Het is mogelijk dat de doelen die hier zijn beschreven en de actuele doelen die Roel Lucassen heeft gedaan kunnen verschillen in verband met deviaties.

De leerplan zal uit twee gedeeltes bestaan:

  • Wie ben ik
  • De doelen

De doelen zijn punten die Roel Lucassen wil onderzoeken of ervaring mee wilt opdoen. Deze doelen hebben te maken met security engineering en een groot deel zal advies zijn van Fontys. De doelen zullen worden beschreven doormiddel van een must/should have lijst met daarnaast additionele kopjes met gerelateerde onderwerpen. 

De planning zal bestaan uit een tijdsch ema waar meerdere doelen in zijn beschreven met daarnaast algemene doelen van Fontys en mogelijke interesses gerelateerd tot security engineering.

Wie ben ik

Mijn naam is Roel Lucassen. Ik ben een derde jaar student aan de Fontys en ik heb gekozen voor security engineering. Ik zat te twijfelen tussen red teaming and security engineering maar dacht toen dat ik security engineering toch wat leuker zou vinden dus heb ik daar voor gekozen.

In de toekomst wil ik iets gaan doen met software bij de politie of het leger. Ik ben nog niet 100% zeker of ik cybersecurity wil doen of software. In ieder geval ben ik meer geïnteresseerd in het voorkomen van cyberleks dan het verdedigen hiertegen. 

De doelen

Hieronder staan interesses, onderwerpen en opdrachten waar ik dit semester aan wil gaan werken.

 

Must have

1.

Hoe schrijf je een verslag en rapporteer dit.

2.

Onderzoek security engineering banen en documenteer

3.

Het maken van een abuse case van de R&D project

4.

Bestudeer en implementeer standaard security principes.

5.

Leer syntax en de opties van regular expressions en pas deze toe op een test-applicatie.

6.

Voer een AES operatie handmatig uit en rapporteer dit. Zie cryptografie voor meer informatie.

7.

Implementeer data encryptie op applicatie level code. Documenteer welke encryptie, waarom de encryptie, waarom niet de andere encryptie, hoe je het hebt geïmplementeerd en een voorbeeld. 

8.

Leer de basis van RSA public cryptography. Maak een document/link naar cryptografie en bestudeer dit en documenteer de vindingen.

9.

Voeg security logging toe aan de applicatie, implementatie en tests.

10.

Maak gebruik van een static code review analysis tool.

11.

Bestudeer de code van een open source app. Voeg hierop static code analyse als handmatige analyse.

12.

Bestudeer een veilige omgeving (server environment bijvoorbeeld) en rapporteer wat je hebt gevonden.

13.

Voeg fuzzy tests toe aan de R&D applicatie.

14.

Apply all security by design steps and techniques in your R&D project:

- gather and discuss security requirements 

- define security threats for the system to develop and perform a risk analysis.

- put the threats in a secure design through misuse cases.

- work out security controls to limit the threats in your misuse cases

- implement the security controls

- test the security controls by developing test cases for the threats. Add fuzzing tests.

15.

Pick an IT application or service to develop to be able to learn and apply all security engineering aspects. Pick an application that needs security and that could be an actual operational environment, e.g. a CTF platform, an information system for a SOC, a web platform thats stores sensitive or confidential data, etc. Give a general description of the application in your portfolio: main functionality, users and user groups, global system architecture. 

Install one or more server in your personal seclab account for development and testing of this very secure application. This server environment will be used for the Red Team - Blue Team event where the security of your application is tested (Red Team) and monitored (Blue Team).
