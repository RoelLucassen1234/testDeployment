Inleiding

Dit document beschrijft wat fuzz testing is, waarom het wordt gebruikt en hoe je het kan toepassen. Er zullen ook een paar voorbeelden worden gegeven.

Wat is Fuzz testing

Fuzz testing beschrijft systeemtestprocessen die een gerandomiseerde benadering omvatten. IT-professionals gebruiken de term vaak om te praten om applicaties te stresstest door willekeurige gegevens in te voeren om eventuele fouten of hang-ups op te sporen. Het idee achter fuzz-testen is dat softwaretoepassingen en -systemen veel verschillende bugs of glitches kunnen hebben met betrekking tot gegevensinvoer en dit zou niet moeten gebeuren

Fuzz-testen kunnen bijvoorbeeld de invoer van verschillende soorten integers, character strings, floats en andere variabelen omvatten die, als ze niet correct worden ingevoerd, ertoe kunnen leiden dat de applicatie vastloopt of crasht.

Een voorbeeld is een integer-veld dat bedoeld is om een paar specifieke getallen te gebruiken zoals een tot en met vijf, maar waar een gebruiker elk getal kan invoeren doordat ze het niet hebben gelimiteerd. Het invoeren van een hoog getal kan dan een fout of crash veroorzaken. Bij fuzz-testen experimenteren ontwikkelaars met het invoeren van veel verschillende soorten willekeurige reacties, en documenteren ze vervolgens eventuele bugs die optreden. In sommige gevallen kunnen ontwikkelaars een tool genaamd fuzzer gebruiken om willekeurige gegevens te injecteren.

Waarom zou je fuzz testing gebruiken

  • Vaak vindt Fuzzy-testen de grote beveiligingsfouten of defecten.
  • Fuzz-testen geven een effectiever resultaat bij gebruik met Black Box Testing, Beta Testing en andere methoden voor foutopsporing.
  • Vaak vindt Fuzzy-testen de grote beveiligingsfouten of defecten.
  • Fuzz-testen is een van de black box-testtechnieken. Fuzzing is een van de meest gebruikte methoden die hackers gebruiken om de kwetsbaarheid van het systeem te vinden.

Hoe fuzz test je

  1. Identify the target system
  2. Identify inputs
  3. Generate Fuzzed data
  4. Execute the test using fuzzy data
  5. Monitor system behavior
  6. Log defects

Fuzz test tools

Tools die worden gebruikt bij webbeveiliging kunnen op grote schaal worden gebruikt bij fuzz-tests zoals Burp Suite, Peach Fuzzer, enz.

Perzik Fuzzer

Peach Fuzzer biedt meer robuuste en beveiligingsdekking dan een scanner. Andere testtools kunnen alleen naar bekende threads zoeken, terwijl Peach Fuzzer gebruikers in staat stelt om bekende en onbekende threads te vinden.

Spike-proxy

Het is een professionele tool die zoekt naar kwetsbaarheden op applicatieniveau in webapplicaties. SPIKE Proxy behandelt de basisprincipes, zoals SQL-injectie en cross-site-scripting, maar het is een volledig open Python-infrastructuur. SPIKE Proxy is beschikbaar voor Linux en Windows.

Webscarab

Webscarab is in Java geschreven en dus overdraagbaar naar veel platforms. Voor het analyseren van applicaties wordt een Webscarab-framework gebruikt dat communiceert via HTTP- en HTTPS-protocollen.

Voorbeeld: Webscarab werkt als een onderscheppende proxy, het stelt de operator in staat om door de browser aangemaakte verzoeken te bekijken en te wijzigen voordat ze door de server worden ontvangen. En laat toe om de reacties die door de server zijn gegenereerd te bekijken en bij te werken voordat ze door de browser zijn ontvangen. Op deze manier, als de web-scarabee een probleem vindt, wordt de lijst met de gemelde problemen gemaakt.

OWASP WSFuzzer

WSFuzzer is een GPL-programma dat is geschreven in Python. GPL heeft een programma momenteel gericht op webservices. In de huidige versie van OWASPWSFuzzer zijn op HTTP gebaseerde SOAP-services het belangrijkste doelwit.
