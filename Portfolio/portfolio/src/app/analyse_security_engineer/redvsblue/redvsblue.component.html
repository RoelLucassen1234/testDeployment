Applicatie

De applicatie had de volgende functies tijdens het eerste evenement: Een gebruiker kon registreren met een email, gebruikersnaam en wachtwoord, een gebruiker kon inloggen als hij een wachtwoord had. Als je ingelogd was kon je als je een admin was alle gebruikers bekijken en verwijderen. De autorizatie werd gedaan doormiddel van JWT tokens. Deze tokens waren gesigned met een RSA 2048 sleutel die random werd gegenereerd.

Voor het deployen maak ik gebruik van ubuntu 2020 server. Op deze server heb ik docker geinstalleerw. Ik bouw hiermee mijn images en die bouw ik via docker-compose. Ik heb ervoor gezorgd dat deze images via github worden gepulled waardoor ik al mijn aanpassingen binnen een minuut kan opzetten

Problemen voor het evenement

Geen goede database beveiliging

Geen ssl certificaat

Geen DOS verdediging

Problemen gevonden door red team

De red teamers hebben geen problemen gevonden met mijn applicatie op het moment. Redenering ervoor was dat er nog niet genoeg op stond. Ze hebben echter niet geprobeerd om toegang te krijgen tot mijn database of ik heb er geen logs over gezien

Problemen die ik heb gevonden tijdens het evenement

Ik heb twee problemen gevonden met de logging. Het eerste probleem was dat ik onnodige informatie (de jwt token) liet zien waardoor ik moeite kreeg met het bekijken van relevante logs. De tweede fout was dat sommige logs niet precies genoeg was voor mijn gevoel en wat ik wat preciezer wil maken

Conclusie

Voor het eerste evenement vind ik dat ik een goede applicatie heb geleverd. Er waren mogelijk iets te weinig features voor de red teamers maar voor de eerste keer vond ik het voldoende. Voor het volgende evenement zou het mogelijk moeten worden om een profiel aan te maken met foto's. hopelijk geeft dit de mogelijkheid voor red teamers om vulnerabilities te vinden.
